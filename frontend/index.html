Technology Stack

Frontend Dashboard

• HTML5 & CSS3: Built using semantic HTML and custom CSS variables for a high-fidelity glassmorphic dark theme.
• Vanilla JavaScript: Handles asynchronous drag-and-drop file uploads, base64 image parsing, and dynamic status updates without heavy framework overhead.
• Interactive Canvas Masking: Implements a double-layer mask using an HTML5 Canvas and dynamic CSS mask-image to generate the cursor-following spotlight reveal effect.
• Intersection Observer: Synchronizes the active state of navigation tabs dynamically based on the current scroll viewport.

Backend API

• FastAPI: High-performance Python web framework used to expose endpoints for real-time model inference and metadata retrieval.
• Uvicorn: Lightning-fast ASGI server implementation used to run the FastAPI application under local environment conditions.
• CORS Middleware: Configured to support seamless frontend-backend communication across ports.
• JSON Manifest System: Serves structured precomputed Colab results (image assets and precalculated graph metrics) using flat JSON files.

Deep Learning Model

• PyTorch: Underlying ML library powering local model loading, tensor manipulation, and GPU/CPU inference.
• U-Net Architecture: State-of-the-art road segmentation network designed with a ResNet-34 encoder pretrained on ImageNet for extracting road masks from satellite imagery.
• Segmentation Models PyTorch (SMP): Library utilized for initializing and configuring the model framework during Colab training.
• Inference Optimization: Features CPU fallback for systems without CUDA capability, executing in just a few seconds per tile.

Graph & Image Processing

• NetworkX: Builds topological road networks (intersections as nodes, road centerlines as edges) to run centrality calculations.
• Scikit-Image (skimage): Performs skeletonization (thinning segmented masks to 1px wide lines) and morphological operations like closing to heal gaps caused by tree cover.
• Vulnerability Scoring: Calculates custom road segment criticality by combining edge betweenness centrality and global connectivity loss metrics.
• OpenCV: Handles image color conversion and image generation before sending the finalized multi-panel comparison visualization to the client.

About Pipeline

Route Resilience

Occlusion‑Robust Road Extraction & Graph‑Theoretic Criticality Analysis for Urban Mobility
Bharatiya Antariksh Hackathon 2026 — ISRO, Problem Statement 4

Overview

Route Resilience is an end‑to‑end pipeline that turns raw satellite imagery into a ranked, resilience‑aware road network. It addresses two problems that are usually treated separately in remote‑sensing literature but are, in practice, the same problem seen from two ends: extraction quality determines whether the resulting road graph is even topologically correct, and graph analysis determines which of the extracted roads actually matter for keeping a city moving.

The problem, in detail

Failure mode one — occlusion. Optical satellite sensors record whatever is in front of them, and quite often that is not the road surface at all. Cloud cover, shadows, and dense tree canopy physically block the sensor's view. A segmentation model trained purely on visible pixels cannot distinguish “there is no road here” from “there is a road here, but I cannot see it”. The result is a road mask riddled with false gaps.

Failure mode two — undifferentiated importance. Even a perfectly extracted road network treats every segment as interchangeable unless you explicitly analyze its role. In reality, a small number of arterial roads and bridges carry a disproportionate share of shortest paths; losing one can isolate entire districts.

System architecture

  • Stage 1 — Segmentation. U‑Net with ResNet34 encoder produces per‑pixel road probability maps.
  • Stage 2 — Occlusion recovery. Fusion with a clean reference pass fills gaps, followed by morphological closing.
  • Stage 3 — Skeletonization. Thins the mask to 1‑pixel centerlines.
  • Stage 4 — Graph construction. Builds an 8‑connected pixel graph, collapses degree‑2 nodes.
  • Stage 5 — Criticality scoring. Combines edge betweenness centrality (70 %) and vulnerability (30 %) into a 0‑1 score.

Training data

The model is trained on 6,400 combined tiles from DeepGlobe Road Extraction and Massachusetts Roads datasets, covering urban, suburban, and rural terrain.

What we validated — including where it struggles

On clear urban roads the pipeline restores missing pixels and maintains connectivity. In dense canopy the model misses many faint roads; lowering the confidence threshold does not recover them, indicating a data‑scarcity limitation.

Why extraction quality and graph analysis cannot be separated

Fragmented extraction yields disconnected graph components, making centrality metrics meaningless and potentially misleading about road importance.
